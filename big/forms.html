Accessibility in Web Design

Forms are an important part of web pages. They allow users to input information so they can send information to the database and get in contact with the company or person owning the website. A lot of people don’t realize the importance of creating a cooperative form. In general, it is best to keep forms simple with the instructions and examples for the user so they can be guided through the form with ease. You should also provide clear error messages and feedback for when the user makes a mistake in their form, such as entering invalid text or forgetting to fill out a question [1].

Forms on web pages are typically created with mostly HTML. HTML is used to create interactive input fields to enter text, checkboxes, radio buttons, and lists with selectable options [2]. The basic parts of a form can be categorized as such:

The form element and its attributes:
The id attribute

Gives your form a unique identifier which can be used to add other programming languages for styling, such as CSS and Javascript.

The method attribute

This specifies how your form will be sent to the processing script. Two types of methods would “GET” and “POST”. POST is more secure.

The action attribute

This specifies which type of script the form will be sent to for processing.


Within the form tag, you can add the following:
The input element, which has the following attributes:
The type attribute

Is used to specify what kind of input will be used. It can be single-line text or etc.

The name attribute

Is used to specify what the input is for

The value attribute

Is used to specify what type of value will be inputted. You could not specify anything and accept any input or you could specify that the value would be text or a radio button or a checkbox or even attaching a photo.

The text-area element

Is used for multi-line text input. The amount of characters, words, or lines can be limited.

The input type="submit" tag

This tag creates a button, which when pressed sends your form.

Accessibility labels

Is used to label the form elements. This is useful for screen readers.

Other additional things that can be added to your form can be the field and legends elements and other programming languages such as CSS and Javascript to style your form [2].
